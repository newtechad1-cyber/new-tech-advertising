# R0.6.2 — Canon Classification & Publishing Population

**Release:** 0.6.2
**Date:** July 6, 2026
**Type:** Architecture + Data Migration

## Mission

Transform NTA from a website into a living educational platform by populating every publishing system with classified knowledge assets and connecting them through a central intelligence layer.

## Architecture Decisions

### ADR-001: PublishingArticle as Single Source of Truth
The `PublishingArticle` entity absorbs the role of `CanonEntry`. No separate CanonEntry entity — the Publishing Engine IS the Canon.

### ADR-002: Data-Driven Everything
Collections, relationships, journeys, and publishing destinations are stored in entities. No hardcoded `canonData.js`. Future content automatically participates without code changes.

### ADR-003: Knowledge Graph Engine
`src/lib/knowledgeGraph.js` is the central intelligence layer. All UI components read from this engine, which queries entities and builds indexes, relationships, and recommendations.

### ADR-004: Homepage Non-Destructive Integration
Homepage layout, messaging, branding, and CTAs are preserved. Dynamic components are available as drop-in replacements for hardcoded content blocks.

## Deliverables

### 1. Publishing Engine Population
- Extended `PublishingArticle` entity with 20+ Canon fields (canon_id, buyer_stage, scores, search intent, relationships)
- Seed data: 58 assets (21 articles, 18 service pages, 11 industry pages, 8 geo pages)
- Admin migration page: `/AdminCanonMigration`

### 2. Journal Population
- JournalIssue records created from all articles
- Cross-references to canon_ids and collections

### 3. YouTube Knowledge Engine Population
- 13 YouTube video records with playlist, article, and service connections

### 4. Canon Relationships
- Every asset has bidirectional relationships via `related_articles`, `related_video_ids`, `related_case_study_ids`, `related_lesson_ids`, `related_services`, `related_industry_slugs`, `related_geo_slugs`
- Knowledge Graph engine resolves all relationships at query time

### 5. Reader Journeys
- `CanonCollection` entity with 10 collections
- Reader progress tracked in localStorage
- Journey navigation with prev/next/progress
- Collection types: reader_journey, topic, series, service_guide, industry_solution

### 6. Internal Linking Engine
- `RelatedContent` component renders related reading, watch next, learn next, etc.
- Two variants: sidebar (vertical) and inline (horizontal cards)
- Suggestions engine identifies unlinked assets with shared themes

### 7. Canonical Management
- Admin page: `/AdminCanonicalManagement`
- 3 duplicate groups identified and marked
- Status tracking: canonical, secondary, redirect, merged

### 8. Homepage Integration (Non-Destructive)
- Drop-in components: `DynamicFeaturedArticles`, `DynamicLatestJournal`, `DynamicCollectionsStrip`, `DynamicCaseStudies`
- Available for homepage integration without redesigning existing layout

### 9. Build Log Automation
- `build_log` asset type added to PublishingArticle schema
- Future releases can auto-create journal entries

### 10. Editorial Dashboard Expansion
- 6 new health widgets: Canon Health, Collection Health, Buyer Stage Distribution, Index Status, Video Coverage, Knowledge Graph Summary
- Available for integration into EditorialDashboard
